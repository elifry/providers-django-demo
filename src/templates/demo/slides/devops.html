What is DevOps?

(and how can we benefit?)
by Eli Fry

Why it matters

  • Fewer breaking changes in builds
  • Lower risk for customers
  • Faster commit to deployment
  • More frequent deployments possible
  • Ship features faster than competitors
  • Less time fixing bugs, more time working on new features

What is DevOps?

  • Cultural philosophies
  • Practices
  • Tools

DevOps Culture

  • Combination of Dev and Ops Teams
  • Breakdown of walls and silos
  • Closer connection to product, shared responsibility
  • Better visibility and communication
  • Hours for feedback, not days or weeks

DevOps Practices

  • Breaking up architecture into smaller parts, decoupling
  • Smaller teams
  • Test and deploy parts separately
  • Less time spent negotiating and coordinating across teams, less meetings
  • Small, frequent commits
  • Builds on every commit
  • Deploy to running environment quickly for further testing
  • All code, infrastructure, and documentation goes in a git repo
  • Continuous Integration (CI)
  • Automated testing, automated builds
  • Continuous delivery (CD)
  • Automated all the way to stage for production
  • Instrastructure as code
  • Scalability
  • Disaster recovery
  • Cloud-ready

DevOps Tools

Pipelines

  • Automated actions and transitions
  • Faster and safer
  • Happier developers

Monitoring

  • Build Logs
  • Find what went into a build
  • Track failures, get notifications
  • Compare performance metrics

DevOps is NOT...

A magical method, role, job description, or group that you can simply plug in and fix all the issues

DevOps is...

The empowering of the developer by using automation, quick feedback, and a growing body of checks and tests to make a safer codebase.

The end goal is...

Selling more robots.


Efficiency in development translates into more features.

DevOps at FANUC so far

  • Git transition, shared repo
  • GitLab, shared issue tracking
  • Newer languages (C# vs VB)
  • Python scripts
  • Build Dashboard (in development)

Going forward

  • Small steps!
  • Gated promotions
  • Automated testing
  • Tracking metrics, feedback loop
